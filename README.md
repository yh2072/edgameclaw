<h2>Why games?</h2>

<p>Not because learning should be <em>"fun."</em></p>

<p>Because <strong>good game design and good pedagogy are the same thing.</strong></p>

<p>A well-designed game level teaches you exactly one thing —<br />
through <strong>failure, repetition, feedback, and mastery</strong> —<br />
in the exact order you need to learn it.</p>

<p><em>That's how humans actually learn.</em></p>

## 📄 &nbsp; Upload anything. Get a playable course.

Textbooks · Research papers · Internal docs · Training manuals · Lecture notes · API references · Wikipedia

Paste your source material → ahafrog reads it, identifies the concepts, matches each one to its natural game mechanic, and generates a full series of playable micro-lessons — with visuals, interactive gameplay, and adaptive pacing.

**No game design experience. No engineering. No curriculum expertise required.**

## How it works

</div>

<br />

**1 →** Paste any learning material — textbook chapter, internal doc, research paper, anything written.

**2 →** ahafrog's AI identifies the core concepts and maps each one to its matching game mechanic.

**3 →** A full playable course is generated — visuals, gameplay, adaptive pacing — ready to publish.

**4 →** Learners play their way to mastery. Every replay deepens the same mental model.
